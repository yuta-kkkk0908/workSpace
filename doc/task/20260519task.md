## 運用指針（固定・再説明不要）
- 正本は `data/investment.db`。`topics/*/inbox/*` は補助ログとして扱う。
- 分析・判断・改善提案は DB 集計を一次根拠とし、ファイルは補助参照に限定する。
- 投資改善は以下のループを前提とする:
  - `backtest -> ルール化 -> entry候補(シグナル評価) -> ルール改善 -> backtest`
- 検証の標準手法は「当時シグナル × 現在ルールの再評価（counterfactual replay）」とする。
- 具体:
  - 過去日の `signals` を固定取得
  - 現在のゲート/ルールで再評価し、当時候補を再構成
  - `backtest_outcomes` と結合して旧運用比（採用率/勝率/DD proxy）を評価
- 「バックテストが足りているか」は上記再評価の母数・品質を判定して答える（感覚では判断しない）。

  - 直近30営業日で旧ロジック比較（trade化率 / T+5 win率 / drawdown proxy）
